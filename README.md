# Problem Solving Forum

코딩 테스트 정복을 위해 좋은 프로그래밍 문제를 공유하고, 기술적인 해결 방법에 대해 자유롭게 토론하는 공동 학습 공간입니다.

## 개요 (Overview)

이 저장소는 개발자들이 서로에게 영감을 주는 문제를 공유하는 열린 토론의 장입니다. 동료 검토(Peer Review)와 다양한 관점의 풀이 공유를 통해 논리적 사고력을 기르고 함께 성장하는 것을 목표로 합니다.

## 참여 방법 (How to Participate)

모든 활동은 **Issues** 탭을 통해 이루어집니다. 참여자는 다음 두 가지 방식으로 기여할 수 있습니다.

### 1. 새로운 문제 공유하기

함께 풀고 싶은 좋은 문제를 발견했다면, 새로운 이슈를 발행하여 공유해 주세요. 작성 시 다음 사항을 준수해야 합니다:

* 문제에 대한 명확한 설명, 제약 사항, 입출력 예시를 포함해야 합니다.
* **필수 사항:** 다른 사람의 재미를 위해 정답이나 본인의 풀이는 반드시 HTML 토글(Toggle) 기능을 사용하여 가려주세요.
* 정답 작성 시 아래 형식을 사용하세요:

<details>
<summary>정답 확인하기</summary>
(이곳에 풀이 설명이나 코드를 작성하세요)
</details>

### 2. 문제 풀이 및 토론하기

다른 사람이 올린 문제를 풀어보고 토론에 참여해 주세요:

* **언어 무관:** 특정 언어의 문법보다는 논리적 흐름과 알고리즘의 효율성에 집중하기 위해, 가급적 **의사코드(Pseudocode)**로 해결 방법을 표현하는 것을 권장합니다.
* **토론:** 더 최적화된 방법이나 다른 시간 복잡도 분석, 혹은 특이한 예외 상황(Edge Case)을 발견했다면 댓글로 자유롭게 의견을 남겨주세요.

## 기여 안내 (Call for Contribution)

이 커뮤니티의 가치는 멤버들의 활발한 참여에 달려 있습니다. 초보자든 숙련된 엔지니어든, 여러분만의 독특한 접근 방식은 다른 누군가에게 큰 도움이 될 수 있습니다. 적극적인 문제 공유와 최적화 제안을 통해 활기찬 학습 환경을 만드는 데 동참해 주세요.
